Power Supply Controller (PSC) Controls Development

  • Contributed to the ALS-U project at LBNL as part of the Osprey DCS team, focusing on the integration and deployment of advanced Power Supply Controller (PSC) systems.
  • Designed and deployed modular EPICS IOCs to control and monitor PSCs, ensuring high-precision power regulation for accelerator magnets and supporting systems.
  • Developed user-facing Phoebus GUIs to provide streamlined visibility and intuitive control of PSC states, interlocks, and diagnostics.
  • Supported power supply integration efforts across BNL and LBNL by standardizing control infrastructure and improving reliability, maintainability, and real-time response.
  • Additionally, created a custom EPICS IOC and Phoebus interface for Tripp Lite PDUs, enabling remote monitoring, outlet-level power cycling, and improved rack-level power management.

Network Infrastructure Upgrades

  • As part of the NSLS-II Data Science and Systems Integration (DSSI) group, I contributed to major network infrastructure improvements supporting accelerator operations.
  • The NSLS-II facility has transitioned to an IaaS (Infrastructure as a Service) model for its network stack. This includes the use of vCenter and Satellite to manage local data centers and virtual servers, providing a flexible and scalable infrastructure to support various applications and services.
  • This virtualization model enables centralized deployment of tools and configurations, improving system efficiency. It also enhances cybersecurity by offering greater control and protection over critical system components.

AI-Assisted Accelerator Timing System

  • Used AI-assisted development to design and implement Ethernet-based EPICS IOCs, replacing deprecated 3100 CPUs and RTEMS. This modernization improved reliability, enabled remote diagnostics, and enhanced system scalability while eliminating reliance on aging VME-based infrastructure.
  • Developed a new device driver for MRF230 series timing modules, leveraging AI-generated code optimizations to streamline implementation and reduce development time. The transition to an Ethernet-based architecture provided greater flexibility in control, remote management, and integration with existing accelerator infrastructure.

EPICS-Based Area Radiation Monitor (ARM) Modernization

  • Updated the EPICS device driver to accommodate new hardware, using AI to optimize the refactoring process.
  • Modernized the way ARMs, part of the Personnel Protection System (PPS), transmit data streams over the network through hardware and configuration upgrades, improving reliability and scalability.

SBM Top-Off Application

Nominal Gaps Application

Beam Dump Recovery Tool
