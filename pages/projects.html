Network Infrastructure Upgrades

  • The NSLS-II facility has transitioned to an IaaS (Infrastructure as a Service) model for its network stack. This includes the use of vCenter and Satellite to manage local data centers and virtual servers, providing a flexible and scalable infrastructure to support various applications and services.
  • This virtualization model enables central deployment of tools and configurations, improving system efficiency. It also enhances cybersecurity by providing better control and protection over the system components.

Systems-Focused Test Benches

  • Designed and implemented test bench environments replicating production configurations, ensuring seamless live spare availability.
  • Enabled groups like Vacuum, RF, and Diagnostics to test upgrades, troubleshoot issues, and validate tools in a controlled, isolated environment.
  • Minimized risks to production systems by keeping test benches completely separate from live environments.
  • Established a standard framework adopted across multiple teams, improving efficiency and reliability.

AI-Assisted Accelerator Timing System

  • Used AI-assisted development to design and implement Ethernet-based EPICS IOCs, replacing deprecated 3100 CPUs and RTEMS. This modernization improved reliability, enabled remote diagnostics, and enhanced system scalability while eliminating reliance on aging VME-based infrastructure.
  • Developed a new device driver for MRF230 series timing modules, leveraging AI-generated code optimizations to streamline implementation and reduce development time. The transition to an Ethernet-based architecture provided greater flexibility in control, remote management, and integration with existing accelerator infrastructure.

Power Supply Controller (PSC) Controls Development

  • Designed and deployed EPICS IOCs for modular PSC systems supporting power supply control infrastructure at both BNL and LBNL.
  • Contributed to the ALS-U project at LBNL by integrating PSC controls that provide precise, reliable power regulation for accelerator environments.
  • Developed a custom EPICS IOC and Phoebus interface for Tripp Lite PDUs, enabling remote monitoring, power cycling, and streamlined management of rack-mounted hardware within accelerator control systems.
  • Developed and supported control system integration to enhance diagnostics, reliability, and maintainability across multiple lab environments.

EPICS-Based Area Radiation Monitor (ARM) Modernization

  • Updated the EPICS device driver to accommodate new hardware, using AI to optimize the refactoring process.
  • Modernized the way ARMs, part of the Personnel Protection System (PPS), transmit data streams over the network through hardware and configuration upgrades, improving reliability and scalability.

SBM Top-Off Application

  • Enhanced machine efficiency by enabling uniform beam injection and implementing ML techniques for optimization.

Nominal Gaps Application

  • Developed a centralized tool allowing operators to control all accelerator insertion devices simultaneously from one location, improving operational efficiency.

Beam Dump Recovery Tool

  • Automated recovery of the machine after a beam dump, ensuring key recovery steps (such as EPS reset and RF state switches) are not overlooked, and the system is configured appropriately.

Injection Configuration Tool

  • Created a Python-based tool that interfaces with EPICS PVs to configure various injection types, simplifying the setup and ensuring better consistency in operations.
